# Accounting Standards

## 1. Tujuan

Dokumen ini menjelaskan posisi Fundara terhadap standar pelaporan keuangan nirlaba.

Fundara tidak ingin menjadi sistem accounting generik. Fundara dirancang sebagai Mission Impact Platform yang memahami bahwa organisasi nirlaba perlu mempertanggungjawabkan sumber daya berdasarkan batasan dana, tujuan sosial, donor reporting, dan laporan keuangan organisasi.

## 2. ISAK 35 untuk Indonesia

Untuk organisasi nirlaba di Indonesia, Fundara mengacu pada **ISAK 35: Penyajian Laporan Keuangan Entitas Berorientasi Nonlaba**.

ISAK 35 menjadi rujukan utama untuk:

- penamaan laporan keuangan;
- penyesuaian istilah aset neto;
- penyajian sumber daya dengan pembatasan dan tanpa pembatasan;
- penyajian laporan yang lebih sesuai dengan karakteristik entitas nonlaba.

Laporan utama yang perlu didukung Fundara:

```text
Laporan Posisi Keuangan
Laporan Penghasilan Komprehensif / Laporan Aktivitas
Laporan Perubahan Aset Neto
Laporan Arus Kas
Catatan atas Laporan Keuangan
```

## 3. FASB ASC 958 / ASU 2016-14

Untuk konteks global, Fundara mengadopsi prinsip konseptual **FASB ASC 958** dan **ASU 2016-14** untuk organisasi not-for-profit.

Fokus utama yang relevan:

- klasifikasi aset neto dengan donor restrictions dan tanpa donor restrictions;
- pelaporan aktivitas organisasi nirlaba;
- transparansi sumber daya yang tersedia dan yang dibatasi;
- hubungan antara donor restriction dan pelaporan keuangan.

Pemetaan konsep:

| FASB / ASC 958 | Fundara |
|---|---|
| Net assets with donor restrictions | Aset Neto Dengan Pembatasan |
| Net assets without donor restrictions | Aset Neto Tanpa Pembatasan |
| Donor restriction | Fund Restriction |
| Release from restriction | Pelepasan Pembatasan Dana |
| Statement of activities | Laporan Aktivitas / Penghasilan Komprehensif |
| Statement of financial position | Laporan Posisi Keuangan |

## 4. Accounting Standard Profile

Fundara perlu menyediakan entitas konfigurasi bernama **Accounting Standard Profile**.

Atribut:

- profile name;
- country;
- reporting framework;
- fiscal year rule;
- default Chart of Account template;
- net asset class mapping;
- report template;
- currency policy;
- status.

Contoh profile:

```text
Indonesia - ISAK 35
Global NFP - ASC 958 Inspired
Custom NGO Reporting
```

## 5. Standards-aware, bukan standards-locked

Fundara harus mampu mendukung standar pelaporan, tetapi tidak mengunci organisasi pada satu format tunggal.

Prinsip:

- standar default disediakan;
- mapping dapat dikonfigurasi;
- nama laporan dapat disesuaikan;
- CoA dapat dimodifikasi;
- report template dapat diperluas;
- local compliance dapat ditambahkan sebagai plugin atau localization pack.
