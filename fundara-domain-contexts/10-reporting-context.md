# Reporting Context

## 1. Ringkasan

**Reporting Context** adalah domain yang mengubah data fund, program, transaksi, evidence, compliance, dan impact menjadi laporan yang dapat digunakan oleh donor, board, manajemen, publik, auditor, dan tim internal.

Dalam Fundara, laporan tidak boleh menjadi pekerjaan manual yang dibangun ulang dari spreadsheet. Laporan harus dihasilkan dari data operasional yang sudah tercatat dengan benar sejak awal.

Reporting Context adalah lapisan akuntabilitas akhir yang memperlihatkan rantai dari dana amanah hingga dampak terukur.

---

## 2. Tujuan Context

Reporting Context bertujuan untuk:

1. Menghasilkan laporan donor, campaign, fund utilization, board, management, audit, business unit, dan impact.
2. Menyediakan report template dan reporting period.
3. Menghubungkan report line dengan transaksi, activity, evidence, dan indicator.
4. Mendukung review, approval, submission, dan archive report.
5. Menghasilkan supporting document register.
6. Menyediakan dashboard akuntabilitas dan performa.

---

## 3. Pertanyaan Domain yang Dijawab

Context ini harus mampu menjawab:

- Laporan apa yang harus dibuat?
- Periode laporan kapan?
- Data apa yang menjadi sumber laporan?
- Apakah laporan sudah direview dan disetujui?
- Apakah angka laporan bisa ditelusuri ke transaksi?
- Apakah evidence untuk transaksi laporan lengkap?
- Apakah laporan sudah dikirim ke donor, board, publik, atau auditor?
- Apakah ada laporan yang overdue?
- Bagaimana posisi fund utilization, budget vs actual, dan impact achievement?

## 7. Workflow Utama

### 7.1 Report Generation

```text
Select Report Type
→ Select Fund / Project / Period
→ Select Report Template
→ Pull Financial Data
→ Pull Activity Data
→ Pull Evidence Status
→ Pull Indicator Achievement
→ Generate Draft Report
```

### 7.2 Report Review and Approval

```text
Draft Report Generated
→ Finance Review
→ Program Review
→ MEAL Review if Impact Data Included
→ Management Approval
→ Mark Approved
```

### 7.3 Report Submission

```text
Approved Report
→ Export / Prepare Submission Package
→ Submit to Donor / Board / Public / Auditor
→ Record Submission Date
→ Attach Submission Evidence
→ Archive Report
```

### 7.4 Supporting Document Register

```text
Select Report
→ List Included Transactions
→ Check Required Evidence
→ Mark Complete / Missing
→ Generate Register
→ Include in Audit Pack or Donor Submission
```

### 7.5 Dashboard Refresh

```text
Schedule Dashboard Refresh
→ Pull Current Fund Data
→ Pull Project Progress
→ Pull Budget vs Actual
→ Pull Compliance Status
→ Pull Impact Achievement
→ Display Role-based Dashboard
```

## 8. Aturan Bisnis

1. Setiap report harus memiliki report type dan reporting period.
2. Donor report harus terhubung ke fund atau grant.
3. Campaign report harus terhubung ke campaign fund.
4. Report line finansial harus dapat ditelusuri ke transaksi.
5. Report line impact harus dapat ditelusuri ke indicator achievement.
6. Laporan tidak boleh submitted jika mandatory review belum selesai.
7. Missing evidence harus terlihat sebelum report approval.
8. Report yang sudah submitted tidak boleh diubah tanpa membuat revision.
9. Submission record harus menyimpan tanggal, pihak penerima, dan metode pengiriman.
10. Dashboard harus role-based agar data sensitif tidak terbuka.

---

## 12. Risiko Desain

Risiko yang perlu dihindari:

1. Laporan hanya menjadi export tabel tanpa traceability.
2. Report line tidak bisa ditelusuri ke transaksi atau evidence.
3. Laporan dibuat manual di luar sistem.
4. Tidak ada review dan approval workflow.
5. Report submitted bisa diubah tanpa revision history.
6. Dashboard menampilkan data sensitif ke role yang salah.
7. Terlalu banyak template di awal sehingga sulit dipelihara.

---

## 13. Prinsip Desain

Reporting Context harus mengikuti prinsip:

> Laporan harus dihasilkan dari kebenaran operasional, bukan direkonstruksi dari spreadsheet.

Fundara harus memastikan bahwa setiap angka, bukti, dan klaim dampak dalam laporan dapat ditelusuri kembali ke fund, activity, transaksi, evidence, dan indicator yang relevan.
